# Game Overhaul Design Spec

## Goal
Transform the current prototype into a 40-60 minute playable Pokemon game with classic gym progression, save system, and polished UX. Target: looks like a year of professional development.

## Success Criteria
- Player can start a new game, progress through 10 maps, defeat 2 gym leaders, and reach an ending
- Save/load works reliably (auto + manual, 3 slots)
- 25 Pokemon species catchable
- 40-60 min first playthrough

---

## Phase 1: Core Infrastructure + Viridian City

### 1A. Save/Load System (`src/save_system.py`)
- JSON-based save format with `"version": 1` field for future migration
- 3 save slots, selectable from main menu "Continue"
- Auto-save on every map transition (threaded write to avoid frame drops)
- Manual save from pause menu
- Atomic writes: write to `.tmp` file, then rename to final path
- Validation on load: clamp HP to max, reject unknown species, fallback to Pallet Town if map ID invalid
- Data saved: player position, map ID, team (species/level/HP/moves/EVs/IVs/status), bag items, badges, pokedex seen/caught, play time, event flags, defeated trainer flags

### 1B. UI Refactor
Split `src/ui.py` (~1900 lines) into focused modules:

| Module | Responsibility | Est. lines |
|--------|---------------|------------|
| `src/ui/__init__.py` | UI manager, state routing | 100 |
| `src/ui/components.py` | Button, HealthBar, TypeBadge, DialogBox | 300 |
| `src/ui/battle_ui.py` | Battle rendering, move select, animations | 450 |
| `src/ui/menu_ui.py` | Main menu, pause, pokemon team, bag | 400 |
| `src/ui/world_hud.py` | World overlay, minimap, location banner | 150 |
| `src/ui/shop_ui.py` | Buy/sell interface | 150 |
| `src/ui/dialog.py` | Dialog box, typewriter, choices | 100 |

Public API (`UI` class) stays identical so `game.py` doesn't change.

**Pre-refactor**: Create visual regression screenshots for each UI state (main menu, starter, battle action, battle moves, pause, pokemon team, bag, world HUD) to verify no rendering regressions after split.

## Phase 2: Battle Depth + Viridian Forest

### 2A. Pokemon Catching
- When "Bag" selected in wild battle, show Poke Balls
- Throw animation: ball arc toward wild pokemon
- Shake animation: 0-3 shakes based on catch rate formula
- **Party full handling**: If party has 6 Pokemon, show message "Your party is full! You can't catch more Pokemon." and block the throw. (No PC storage in scope)
- Catch success: pokemon added to party, pokedex registered
- Catch fail: ball breaks, turn consumed
- Catch rate: `(3*maxHP - 2*currentHP) * catchRate * ballMod / (3*maxHP)`

### 2B. Evolution System (`src/evolution.py`)
- Level-based evolution triggers checked after EXP gain
- Evolution screen: old sprite â†’ white flash (~2 sec) â†’ new sprite, name update
- **Cancel window**: During the white flash, show "Press B to cancel" text. If B pressed, evolution cancelled. Re-triggers on next level-up.
- Evolution data in species definition: `"evolves_to": {"species_id": 6, "level": 36}`
- Evolutions in scope: Charmanderâ†’Charmeleon(16)â†’Charizard(36), Bulbasaurâ†’Ivysaur(16), Squirtleâ†’Wartortle(16), Caterpieâ†’Metapod(7)â†’Butterfree(10), Pidgeyâ†’Pidgeotto(18), Weedleâ†’Kakuna(7)â†’Beedrill(10)

### 2C. Trainer AI Tiers

| Tier | Used By | Move Selection | Items | Switching |
|------|---------|---------------|-------|-----------|
| Basic | Route trainers, Bug Catchers | Random from available moves | None | Never |
| Smart | Gym Leaders, Rival | Prefer super-effective moves; avoid not-very-effective | 2 Potions, use when HP < 30% | Switch when at type disadvantage |

Smart AI knows player Pokemon's types but not specific moves.

### 2D. Content - Viridian Forest

## Phase 3: Gyms + Story

### 3A. Event/Cutscene System (`src/cutscene.py`)
JSON-based cutscene scripts. Initial scope limited to 4 action types:

```json
[
  {"action": "dialog", "speaker": "Prof. Oak", "text": "Welcome to the world of Pokemon!"},
  {"action": "fade", "direction": "out", "duration": 1.0},
  {"action": "wait", "duration": 0.5},
  {"action": "battle", "trainer_id": "rival_1"}
]
```

Camera movement and NPC pathing deferred to Phase 4 polish. Player can skip cutscenes by holding ESC (3 second hold). Malformed scripts log a warning and skip the bad action.

### 3B. Badge System
- Badge data in player save (list of acquired badge IDs)
- Badge get fanfare: flash + text "You got the Boulder Badge!"
- Badges viewable in Trainer Card (pause menu submenu)
- Badges are progression markers (no gameplay effects)

### 3C. BGM/SE System
- OGG format only (reliable cross-platform in Pygame)
- Optional: graceful fallback to silent mode if files missing
- Track categories needed (8 tracks):
  1. Title screen
  2. Town theme
  3. Route theme
  4. Battle (wild)
  5. Battle (trainer/gym)
  6. Victory fanfare
  7. Evolution
  8. Pokemon Center heal
- Source: CC0/free chiptune packs (opengameart.org) or contributor-provided. Mark as "nice to have" â€” game fully playable silent.

### 3D. Content - Pewter City + Gym
- **Route 2 map** (20x40): connects Viridian Forest to Pewter City
- **Pewter City map** (35x35): Gym, Pokemon Center, Museum (exterior only), houses
- **Pewter Gym**: Rock-type, Gym Leader Brock
  - Brock's team: Geodude Lv.12 (Tackle, Rock Throw), Onix Lv.14 (Rock Throw, Bind, Harden)
  - Pre-battle cutscene, post-victory: Boulder Badge + TM (placeholder)
- **Rival battles**:
  - Viridian City entrance (rival: Lv.8 starter + Lv.6 Pidgey) â€” Basic AI
  - Viridian Forest exit (rival: Lv.14 starter + Lv.12 Pidgeotto) â€” Smart AI
- **Oak's Lab intro**: Opening cutscene (dialog-only), receive Pokedex item

- Day/night gameplay effects (visual tint stays, no spawn changes)
- Full 151 Pokemon
- TM/HM system (badges don't unlock moves)
